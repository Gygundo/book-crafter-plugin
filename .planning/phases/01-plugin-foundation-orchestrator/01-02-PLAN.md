## 3. Pipeline State Detection Section
Scan the project directory to determine current pipeline state. This is the resume logic.

**Detection algorithm:**

```
1. Check for output/*.docx -> if exists, pipeline is COMPLETE
2. Check for edited/ch*-final.md -> if count matches outline chapter count, Stage 4 COMPLETE
3. Check for drafts/ch*-draft.md -> if count matches outline chapter count, Stage 3 COMPLETE
4. Check for research/ch*-research.md -> if count matches outline chapter count, Stage 2 COMPLETE
5. Check for chapter-outline.md with <!-- APPROVED --> marker -> Stage 1 COMPLETE
6. Check for chapter-outline.md without approval marker -> Stage 1 IN PROGRESS (needs approval)
7. None of the above -> Pipeline not started
```

**Partial completion handling:**
- If a stage directory exists but file count doesn't match the outline's chapter count, the stage is PARTIALLY COMPLETE
- Read `chapter-outline.md` to determine expected chapter count (count the chapter entries)
- Report which specific chapters are missing (e.g., "Research: 8/12 chapters complete, missing ch09, ch10, ch11, ch12")

**Chapter count extraction:**
To determine expected chapter count, read `chapter-outline.md` and count lines matching the pattern `## Chapter` or equivalent chapter heading markers.

## 5. Stage Execution Section
When executing the next stage:

1. **Identify the next stage** based on state detection
2. **Check if the stage skill is implemented** -- read the skill's SKILL.md and check for `[STUB` marker. If the skill is a stub, tell the user: "Stage [N]: [Name] is not yet implemented (coming in Phase [X]). The pipeline cannot proceed past this point until that phase is built."
3. **For implemented stages**, invoke the appropriate skill with the project directory path
4. **After stage completion**, re-scan pipeline state and display updated dashboard
5. **Stage transitions**: The output of one stage becomes the input of the next. The orchestrator does not transform data between stages -- it simply tells the next stage where to find its input files.

### Stage-Specific Orchestration Notes

**Stage 1 (Outline):** After the outliner produces `chapter-outline.md`, present it to the user for review. The user must explicitly approve before proceeding. On approval, add `<!-- APPROVED -->` marker to the file. On rejection, allow the user to provide feedback and re-run the outliner.

**Stage 3 (Write):** Spawn chapter-writer subagents in parallel using the Agent tool. Batch in waves of 8-10 if the book has more chapters than the parallelism cap. Each agent receives:
- Book DNA path: `[project]/book-dna.md`
- Chapter assignment: chapter number, title, outline from chapter-outline.md
- Research path: `[project]/research/ch[NN]-research.md`
- Output path: `[project]/drafts/ch[NN]-draft.md`

**Stage 4 (Edit):** The editor reads all drafted chapters. For books with more than 15 chapters, use the chapter-editor subagent with a rolling window (current chapter + 1 chapter overlap on each side).

## 6. Execution Modes Section

### Mode 1: Guided (Default)
Show the dashboard, explain what the next stage does, and ask for confirmation before proceeding.

### Mode 2: Full Pipeline ("write the whole book")
When the user says "write the whole book", "full pipeline", or "run everything":
1. Create project if needed
2. Execute each stage in sequence
3. Pause at approval gates (outline review)
4. Report progress after each stage
5. Stop at the first stub skill encountered

### Mode 3: Resume ("continue", "resume", "pick up where we left off")
1. Detect pipeline state
2. Show dashboard
3. Offer to continue from the next incomplete stage

### Mode 4: Status Only ("book status", "where am I")
Just show the dashboard. Don't execute anything.

## 7. Error Handling Section

- **Project directory not found**: "No book project found at ~/Documents/Books/[name]. Would you like to create a new project?"
- **Outline not approved**: "The outline hasn't been approved yet. Please review it before proceeding to research."
- **Stub skill encountered**: "Stage [N] ([skill name]) is not yet implemented. It will be available in Phase [X]. The pipeline pauses here."
- **Partial stage completion**: "Stage [N] is partially complete ([x]/[n] chapters). Would you like to resume and complete the remaining chapters?"
- **Missing dependencies**: If a stage's input files are missing (e.g., trying to write without research), report which files are expected and missing.
