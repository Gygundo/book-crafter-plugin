**# Book Outliner**

Brief intro: "Generates a chapter-by-chapter book outline with structured metadata per chapter, designs a narrative arc, and populates the Book DNA master context document after user approval."

**## 1. On Invocation**

Receive the project directory path via `$ARGUMENTS`. Read the following from the project directory:
1. `book-dna.md` -- for metadata (title, subtitle, author, size tier, target word count)
2. `voice-profile.md` -- for voice characteristics to inform outline tone

Extract from book-dna.md:
- Title, Subtitle, Author
- Size tier (booklet | short | standard)
- Topic brief or description (from Metadata or provided inline by orchestrator)

**## 2. Determine Mode**

Check the project directory for source content:
- If a `sources/` directory exists and contains `.md`, `.txt`, or `.docx` files: use **Source Ingestion Mode** (section 4)
- If no source files are present: use **Topic Brief Mode** (section 3)

Log which mode is being used: "Mode: Topic Brief" or "Mode: Source Ingestion ([N] source files found)"

**## 3. Topic Brief Mode**

Generate a book outline from scratch using the topic brief, key themes, target audience, and optional scriptures from the project metadata.

Instructions (include all of these in the SKILL.md):

1. **Determine chapter count** from size tier:
   - Booklet: 5-8 chapters, total 15-20K words, ~2,500-3,500 words/chapter
   - Short: 8-12 chapters, total 15-25K words, ~1,800-2,500 words/chapter
   - Standard: 12-20 chapters, total 40-60K words, ~3,000-4,000 words/chapter

2. **Design the narrative arc** before writing individual chapters:
   - Opening: What tension or question draws the reader in?
   - Progressive revelation: How does understanding deepen chapter by chapter?
   - Climax: What is the peak revelation or turning point?
   - Resolution: How does the book land -- what does the reader walk away with?

3. **Assign momentum positions** to each chapter:
   - Foundation (chapters 1-2): Establish the premise, hook the reader, lay groundwork
   - Building (early-middle chapters): Develop arguments, introduce complexity, deepen
   - Accelerating (mid-late chapters): Increase intensity, connect threads, raise stakes
   - Climax (1-2 chapters near end): Peak revelation, the "aha moment", most powerful content
   - Landing (final 1-2 chapters): Resolution, application, send-off with lasting impact

4. **For each chapter, generate:**
   - Title (compelling, not generic)
   - Hook strategy: Choose ONE of [Bold declaration, Rhetorical question, Counter-intuitive claim, Tension-creating observation] and write the specific hook in 1-2 sentences
   - Core argument: The single central claim this chapter makes (1 sentence)
   - Key arguments: 3-5 supporting arguments that build the core argument
   - Supporting scriptures: 2-5 scripture references relevant to the chapter's argument (for theological books; omit for non-theological voice profiles)
   - Momentum position: Foundation | Building | Accelerating | Climax | Landing
   - Connects to: Which other chapters this chapter foreshadows, builds on, or callbacks to

5. **Cross-chapter coherence check:** After generating all chapters, review the complete outline for:
   - Does each chapter build on the previous one?
   - Are there gaps in the argument progression?
   - Do the momentum positions actually escalate?
   - Are there enough cross-chapter connections (foreshadowing, callbacks)?
   - Does the climax chapter deliver on the tension set up in chapter 1?

**## 4. Source Ingestion Mode**

Generate a book outline from existing content (sermon transcripts, notes, blog posts, outlines).

Instructions (include all of these):

1. **Read all source files** from the `sources/` directory
2. **Extract themes and arguments**: Identify the major themes, recurring arguments, key quotes, and structural patterns across all source material
3. **Critical instruction: Do NOT mirror the source structure.** A sermon series and a book have fundamentally different rhythms. Sermons are standalone talks; a book is a progressive argument. A 6-sermon series should NOT automatically become 6 chapters. Instead:
   - Group related themes across multiple source files
   - Identify the natural progressive argument that emerges
   - Design a NEW book structure that serves the reader's journey
4. **Apply the same chapter generation process** as Topic Brief Mode (section 3, steps 1-5), but informed by the extracted themes and arguments
5. **Preserve key quotes and illustrations** from source material: Note specific quotes, illustrations, or examples from the source that should appear in specific chapters. Include these as a "Source Material Notes" bullet under each chapter.

**## 5. Output: chapter-outline.md**

Write the outline to `[project]/chapter-outline.md` in this exact format:

```markdown
# Book Outline: [Title]

## Book Arc
[Opening tension] -> [Progressive revelation] -> [Climactic truth] -> [Resolution]

## Size Tier
[booklet | short | standard]
Target: [total word count] words, [chapter count] chapters, ~[per-chapter words] words/chapter

## Chapter 1: [Title]
- **Hook strategy:** [Type] -- [Specific hook description in 1-2 sentences]
- **Core argument:** [Single sentence central claim]
- **Key arguments:**
  1. [Argument 1]
  2. [Argument 2]
  3. [Argument 3]
- **Supporting scriptures:** [Scripture 1], [Scripture 2], ...
- **Momentum position:** [Foundation | Building | Accelerating | Climax | Landing]
- **Connects to:** Ch [X] (foreshadows...), Ch [Y] (builds on...)
- **Target word count:** ~[N] words

## Chapter 2: [Title]
...
```

Every chapter MUST have all fields. The "Supporting scriptures" field may be "[N/A -- non-theological voice profile]" if the voice profile does not have a Theological/Domain Framework section.

**## 6. Post-Approval: Generate Book DNA**

This section executes ONLY after the orchestrator confirms the user has approved the outline (the orchestrator adds `<!-- APPROVED -->` to chapter-outline.md and then re-invokes the outliner for Book DNA generation, OR this step runs as part of the same invocation after the orchestrator signals approval).

Read the following:
1. `[project]/chapter-outline.md` (the approved outline)
2. `[project]/voice-profile.md` (the voice profile)
3. `${CLAUDE_PLUGIN_ROOT}/references/book-dna-template.md` (the template)

Populate `[project]/book-dna.md` by filling in every section of the template:

1. **Metadata**: Already partially populated by the orchestrator. Update Chapter count and Target word count from the outline's Size Tier section.

2. **Voice Profile**: Copy the following sections from `voice-profile.md`:
   - Tone -> Voice Profile > Tone
   - Sentence Patterns -> Voice Profile > Sentence Patterns
   - Vocabulary (Use and Avoid) -> Voice Profile > Vocabulary
   - Emphasis Techniques -> Voice Profile > Emphasis Techniques

3. **Theological/Domain Framework**: Copy from `voice-profile.md` if present. If the voice profile has no Theological/Domain Framework section, write "[No domain framework specified -- use general knowledge]".

4. **Book Arc**: Copy the Book Arc line from chapter-outline.md.

5. **Chapter Map**: Build a table from the outline:
   | Ch | Title | Core Argument | Opening Hook Strategy | Key Scriptures | Connects To |
   One row per chapter, data extracted from the outline.

6. **Running Themes**: Analyse the outline and identify 3-7 themes that recur across multiple chapters. For each theme, note which chapters introduce, develop, and climax it.

7. **Key Terms and Jargon**: Extract key terms from the outline that need consistent definition across chapters. Include the term, a brief definition, and which chapter first uses it.

8. **Cross-Chapter Continuity**: Based on the "Connects to" fields in the outline, write explicit continuity notes:
   - Foreshadowing: "Chapter 2 sets up [concept] that pays off in Chapter 7"
   - Callbacks: "Chapter 10 references back to the illustration from Chapter 3"
   - Running metaphors: Any metaphor that should recur across chapters
   - Recurring imagery: Visual or conceptual imagery that builds throughout

9. **Style Rules**: Derive from voice profile and outline:
   - Spelling convention (default: British/SA English)
   - Scripture translation default (from voice profile's Scripture Handling, or NKJV if not specified)
   - Target words per chapter (from outline's Size Tier)
   - Any formatting rules from the voice profile

10. **Add READ-ONLY marker** at the very top of book-dna.md, before the title:
```
<!-- READ-ONLY: Do NOT modify this document during parallel chapter generation. Updates happen between pipeline stages only. -->
```

After writing book-dna.md, confirm completion: "Book DNA generated. [N] chapters mapped, [M] running themes identified, [K] key terms defined."

- Source Ingestion Mode must TRANSFORM source structure, not mirror it.

- Per-chapter word targets must be calculated from the size tier, not left as placeholders.
- Book DNA generation happens ONLY after outline approval. If the outline is not approved, do not generate Book DNA.
- The Book DNA READ-ONLY marker is critical for VOICE-06 compliance. Never omit it.
