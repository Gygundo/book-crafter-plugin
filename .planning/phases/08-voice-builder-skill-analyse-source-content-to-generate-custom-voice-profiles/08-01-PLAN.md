Create `skills/voice-builder/SKILL.md` with the following structure. This is a large skill file -- expect 250-350 lines of markdown.

**Frontmatter (per D-12 standalone + orchestrator-integrated):**
Use YAML frontmatter with these fields:
- name: voice-builder
- description: "Analyse a directory of markdown files to generate a custom voice profile. Can be invoked standalone or through the orchestrator during voice selection. Triggers on: 'build voice profile', 'analyse my writing', 'extract voice', 'voice from my content', 'generate voice profile'."
- user-invocable: true
- allowed-tools: Read, Write, Bash, Grep, Glob

**Section 1: Overview** -- Explain the skill's purpose: analyse .md files from a user-provided directory, extract voice characteristics via two-pass analysis, generate a profile conforming to voice-profile-spec.md. Both standalone and orchestrator-integrated (per D-12).

**Section 2: On Invocation** -- Accept input from user:
- Required: directory path containing source material (per D-02)
- Optional: preferred profile name (otherwise auto-generated per D-10)
- Per D-01: Only .md files. If user provides non-.md path, inform them to convert first.

**Section 3: Corpus Assessment** -- Before analysis:
1. Use Glob to find all `**/*.md` files recursively in the provided directory (per D-02)
2. Use Bash to count total files and total word count: `find [dir] -name "*.md" -type f | wc -l` and `find [dir] -name "*.md" -type f -exec cat {} + | wc -w`
3. Apply confidence tiers (per D-03):
   - HIGH: 10,000+ words AND 5+ files -- proceed without warnings
   - MEDIUM: 5,000-10,000 words OR 3-4 files -- warn user: "Your corpus is below the recommended size (5+ files, 10,000+ words). I'll proceed but some profile sections may be marked <!-- INFERRED --> where confidence is lower. For best results, add more source material."
   - LOW: Under 5,000 words OR 1-2 files -- warn user: "Your corpus is quite small. All profile sections will be marked <!-- INFERRED -->. I recommend adding more source material for a more accurate profile."
4. Per D-04: Analyse ALL .md files. No filtering of short files, outlines, or rough notes.

**Section 4: Pass 1 -- Statistical Extraction (per D-05, D-06)**

Read source files in batches of 3-5 files at a time to manage context. For each batch, extract patterns across four categories.

**Category 1 -- Sentence Patterns and Rhythm:**
- Count average sentence length (word count) across sampled paragraphs
- Measure sentence length distribution: percentage of short (<8 words), medium (8-20 words), long (20+ words)
- Note fragment usage frequency (sentences without a verb, under 5 words)
- Count rhetorical questions per 1000 words
- Identify repetition patterns: anaphora (repeated sentence starts), epistrophe (repeated endings), parallel structures
- Note intensity building patterns: do paragraphs escalate? Do sections build to punchy conclusions?

**Category 2 -- Vocabulary and Word Choice:**
- Identify the 15-20 most frequent distinctive words (excluding common stop words: the, a, is, it, etc.)
- Identify characteristic phrases: 2-3 word combinations that recur across multiple files
- Assess register: formal (passive voice, complex clauses, hedging) / conversational (contractions, direct address, fragments) / mixed
- List domain-specific terminology (theological terms, leadership jargon, technical vocabulary)
- Note notable absences: words or constructions the author never uses despite opportunities

**Category 3 -- Tone and Emotional Register:**
- Directness: ratio of declarative statements vs hedged/qualified statements
- Emotional warmth: frequency of personal pronouns (I, we, you), vulnerability markers, personal anecdotes
- Authority posture: how often does the author assert vs explore vs question?
- Formality spectrum: contractions, slang, colloquialisms vs formal diction
- Humour/wit: present or absent, frequency, type (self-deprecating, observational, wordplay)

**Category 4 -- Structural Patterns:**
- Average paragraph length (sentences per paragraph)
- Argument building: deductive (claim then evidence) / inductive (evidence then claim) / narrative (story then insight)
- Transition patterns: how does the author move between ideas? Explicit connectors vs implicit flow?
- Story/anecdote frequency: how often are personal stories, illustrations, or examples used?
- Emphasis techniques: bold text, italics, repetition, short standalone sentences, exclamation marks

After processing all batches, aggregate findings into a structured intermediate summary (held in working memory, not written to disk). This summary is the input for Pass 2.

**Section 5: Pass 2 -- Profile Synthesis (per D-05)**

Using Pass 1 findings, generate each voice profile section. Every claim must be backed by evidence from Pass 1. The output file structure must be:

  # Voice Profile: [Auto-Generated Name]
  > Built from [N] source files ([M] words) in [directory path].
  > Generated by book-crafter:voice-builder on [date].
  <!-- Validated against voice-profile-spec.md. All required sections (1-5) present. -->

Then generate these sections with evidence-backed content:

- **## Tone** -- Synthesise from Category 3 findings. Use specific evidence: "Bold and direct -- 73% of statements are declarative with no hedging. Warm -- personal pronouns appear every 2-3 sentences."
- **## Sentence Patterns** -- Synthesise from Category 1. Include specific numbers: "Average sentence length: 14 words. Fragment usage: 18% of sentences under 5 words."
- **## Vocabulary > ### Use** -- From Category 2: list 10-15 characteristic words and phrases with examples from source.
- **## Vocabulary > ### Avoid** -- Infer from what the author DOESN'T do. At least 5 items.
- **## Emphasis Techniques** -- From Category 1 + 4: 4-6 techniques with brief descriptions and source examples.
- **## Anti-Patterns (Never Do This)** -- 4-6 guardrails based on what the source voice is NOT.
- **## [Domain] Framework** (optional, per D-07) -- Only if domain detected and user confirmed.
- **## Calibration Examples** (per D-08):
  - **### Target Quality** -- 3 passages (100-200 words each) from source material, verbatim, cited as "From [filename]". Each demonstrates different aspect (tone, rhythm, structure).
  - **### What to Avoid** -- 3 synthetic counter-examples (100-150 words each) labelled "WRONG -- [drift type]": academic drift, generic AI drift, opposite-tone drift.

If corpus confidence is MEDIUM: mark sections with insufficient evidence (pattern in fewer than 3 files) with `<!-- INFERRED -->`.
If corpus confidence is LOW: mark ALL sections with `<!-- INFERRED -->`.

**Section 6: Domain Framework Auto-Detection (per D-07)**

Before presenting the review summary, scan Pass 1 findings for domain signals:

- **Theological:** Scripture references (book chapter:verse patterns), theological terms (grace, covenant, faith, salvation, kingdom, redemption, sanctification), prayer language. Threshold: 5+ theological terms across 3+ files.
- **Leadership:** Management terminology (team, strategy, vision, execution, culture, hire, scale). Threshold: 5+ terms across 3+ files.
- **Self-help:** Personal development language (habits, mindset, growth, goals, routines). Threshold: 5+ terms across 3+ files.
- **Teaching/Academic:** Citation patterns, formal argument structure, hedging language, passive voice >30%.
- **Conversational/Memoir:** First-person narrative dominance (>50% paragraphs start with "I"), personal anecdotes.

Present detection to user: "I detected a [domain] framework in your writing (based on [evidence]). Should I include a [Domain] Framework section? You can also specify a different domain or skip."

**Section 7: Auto-Naming (per D-10)**

1. Take detected domain (if any): "pastoral", "leadership", "teaching", "memoir", etc.
2. Take primary tone descriptor from Category 3: "conversational", "bold", "reflective", "direct", "warm"
3. Combine: `{domain}-{tone}.md` or just `{tone}.md` if no domain
4. Sanitise: lowercase, hyphens only, no special characters
5. Examples: `pastoral-teaching.md`, `leadership-conversational.md`, `bold-narrative.md`

**Section 8: Review Gate (per D-09, mirrors outliner approval pattern)**

Present review summary showing: corpus stats (files, words), confidence level, suggested filename, detected voice characteristics (tone summary, sentence metrics, vocabulary register, top emphasis techniques, detected domain, any INFERRED sections).

Three options:
1. **Approve** -- Save profile to `${CLAUDE_PLUGIN_ROOT}/references/voice-profiles/[name].md` (per D-13). Confirm to user.
2. **Adjust** -- Apply requested changes, re-present summary.
3. **Regenerate** -- Re-run from Section 3.

Per D-11: Always create fresh. If same name exists, append number: `pastoral-teaching-2.md`.

**Section 9: Constraints and Reminders**
- Per D-01: Markdown only. No .docx, .txt, .pdf parsing.
- Per D-04: Analyse ALL .md files. No filtering.
- Per D-11: New profile only. Never merge/update existing.
- Authenticity over polish: reflect ACTUAL voice, not idealised.
- Every section must cite specific evidence from corpus. No generic descriptions.
- If contradictory signals found (formal + casual files), note range in review and let user decide.
- For very large directories (100+ files): process in batches, aggregate. Never read all at once.
