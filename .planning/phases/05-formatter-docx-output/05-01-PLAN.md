Replace the stub in `skills/formatter/SKILL.md` with a complete skill definition. Preserve the existing frontmatter (name, description, user-invocable: false) but add `allowed-tools: Read, Write, Bash, Glob, Grep`.

The skill must define the following sections:

**Section 1: Overview**
- Purpose: convert all `edited/ch[NN]-final.md` files into a single professional .docx
- Input: project directory path (receives from orchestrator)
- Output: `output/[Book Title].docx`
- Prerequisites: all `edited/ch[NN]-final.md` files exist, `book-dna.md` exists

**Section 2: Pre-flight Checks**
When invoked, the formatter must:
1. Verify docx is available: `node -e "require('docx')" 2>/dev/null || npm install -g docx`
2. Read `book-dna.md` from the project directory, extract: Title, Subtitle (may be empty), Author, Chapter count, Key Terms table, Style Rules
3. Count `edited/ch[NN]-final.md` files using `ls edited/ch*-final.md | wc -l`, verify count matches book-dna chapter count
4. Read `voice-profile.md` for any spelling convention rules (British/US)

**Section 3: Markdown Parsing**
Define a `parseInlineFormatting(text)` function that splits paragraph text into TextRun objects:
- `**text**` becomes `new TextRun({ text, bold: true })`
- `*text*` becomes `new TextRun({ text, italics: true })`
- Plain text becomes `new TextRun(text)`
- Handle nested: `***text***` = bold + italic
- Smart quote conversion: replace straight `"` with unicode left/right double quotes (`\u201C` / `\u201D`) based on position, straight `'` with unicode left/right single quotes (`\u2018` / `\u2019`) based on position

Define a `parseChapterMarkdown(content)` function that:
- Strips the `<!-- METADATA ... -->` block (but extracts `scriptures_used` for the scripture index)
- Splits content on blank lines to get paragraphs
- Skips the first line if it matches `# Chapter N:` pattern (heading handled separately)
- Returns an array of Paragraph objects using parseInlineFormatting for each paragraph
- Uses style "Normal" with Georgia font, 12pt (size: 24 in half-points), 1.5 line spacing (line: 360), 6pt after (after: 120)

**Section 4: Document Styles**
Define the complete styles object for the Document:
```javascript
styles: {
  default: {
    document: {
      run: { font: "Georgia", size: 24 }, // 12pt body
    },
  },
  paragraphStyles: [
    {
      id: "Heading1", name: "Heading 1",
      basedOn: "Normal", next: "Normal", quickFormat: true,
      run: { size: 48, bold: true, font: "Georgia" }, // 24pt chapter titles
      paragraph: {
        spacing: { before: 480, after: 240 },
        outlineLevel: 0, // REQUIRED for TOC pickup
      },
    },
    {
      id: "Normal", name: "Normal",
      run: { font: "Georgia", size: 24 }, // 12pt
      paragraph: {
        spacing: { line: 360, after: 120 }, // 1.5 line spacing, 6pt after
      },
    },
  ],
}
```

**Section 5: Document Section Architecture**

Define each section of the document in order:

**Section 5a: Half Title Page** (FMT-06)
- Single section, no headers, no footers
- Centered book title only, pushed down ~3 inches (spacing before: 4320)
- Georgia 36pt (size: 72), bold
- Page size: 12240 x 15840 DXA (US Letter), margins: top 2160, right 1440, bottom 2160, left 1440

**Section 5b: Full Title Page** (FMT-02, FMT-06)
- No headers, no footers
- Centered: book title (Georgia 36pt bold), subtitle if present (Georgia 18pt italic, 14pt = size: 36), author name (Georgia 14pt = size: 28)
- Spacing between elements: title pushed down ~2.5 inches (before: 3600), subtitle 480 after title, author 960 after subtitle

**Section 5c: Copyright Page** (FMT-06)
- No headers, no footers
- Left-aligned, Georgia 10pt (size: 20)
- Content lines (each a separate Paragraph):
  - `Copyright (c) [year] [Author Name]`
  - `All rights reserved.`
  - blank paragraph
  - `No part of this publication may be reproduced, distributed, or transmitted in any form or by any means without the prior written permission of the author.`
  - blank paragraph
  - `Unless otherwise indicated, all Scripture quotations are taken from the New King James Version (R). Copyright (c) 1982 by Thomas Nelson. Used by permission. All rights reserved.` (only if voice profile indicates theological content or scripture translation is set in style rules)
  - blank paragraph
  - `ISBN: [To be assigned]`
  - `First edition, [year]`

**Section 5d: Dedication Page** (FMT-06)
- No headers, no footers
- Centered, italic, Georgia 14pt (size: 28)
- Content: `[Dedication to be added]` -- placeholder since Book DNA does not include dedication text
- Pushed down ~3 inches

**Section 5e: Table of Contents** (FMT-03, FMT-06)
- Roman numeral page numbers in footer (NumberFormat.LOWER_ROMAN, start: 1)
- Footer: centered page number only (no "Page X of Y")
- Content:
  - Paragraph with heading: HeadingLevel.HEADING_1, text "Contents" (but do NOT use pageBreakBefore since this is a new section)
  - `new TableOfContents("Table of Contents", { hyperlink: true, headingStyleRange: "1-1" })`
- CRITICAL: Document must set `features: { updateFields: true }` for TOC to auto-populate

**Section 5f: Body Section -- All Chapters** (FMT-01, FMT-04, FMT-05)
- ONE section containing ALL chapters (chapters separated by pageBreakBefore on heading)
- Arabic page numbers restart at 1: `pageNumbers: { start: 1, formatType: NumberFormat.DECIMAL }`
- Header: centered book title, Georgia 9pt (size: 18), italic
- Footer: centered "Page X of Y" using `PageNumber.CURRENT` + `PageNumber.TOTAL_PAGES`, Georgia 9pt (size: 18)
- For each chapter:
  - Heading paragraph: `heading: HeadingLevel.HEADING_1`, `pageBreakBefore: true`, text is chapter title from book-dna Chapter Map
  - Followed by parsed chapter content paragraphs from parseChapterMarkdown

**Section 5g: Back Matter -- About the Author** (FMT-07)
- Continues arabic page numbering (no restart)
- New section with pageBreakBefore on heading
- Heading: "About the Author" (HeadingLevel.HEADING_1, pageBreakBefore: true)
- Content: check book-dna.md for an `author_bio` field in Metadata. If empty or absent, use placeholder: `[Author bio to be added]`

**Section 5h: Back Matter -- Scripture Index** (FMT-07)
- Heading: "Scripture Index" (HeadingLevel.HEADING_1, pageBreakBefore: true)
- Auto-extraction process:
  1. Read each `edited/ch[NN]-final.md`
  2. Extract from METADATA block `scriptures_used` field
  3. Also regex-scan chapter body with: `/(?:\d\s+)?[A-Z][a-z]+(?:\s+[A-Z][a-z]+)?\s+\d+:\d+(?:-\d+)?/g`
  4. Build Map of scripture reference -> Set of chapter numbers
  5. Sort by canonical Bible book order using BIBLE_BOOKS array (66 books, Genesis through Revelation)
  6. Format as paragraphs: one line per reference, format: `[Reference] .... Chapter [N], [M]` (using tab stops with dot leader for professional look)
- If no scripture references found (non-theological book), omit this section entirely

**Section 5i: Back Matter -- Glossary** (FMT-07)
- Heading: "Glossary" (HeadingLevel.HEADING_1, pageBreakBefore: true)
- Source: Book DNA "Key Terms and Jargon" table
- Format as a table with two columns: Term | Definition
  - Table width: 9360 DXA (content width with 1" margins on US Letter)
  - Column widths: [2800, 6560] DXA (must sum to 9360)
  - Cell widths match column widths
  - Header row with light blue shading (fill: "D5E8F0", type: ShadingType.CLEAR)
  - Body rows with borders (BorderStyle.SINGLE, size: 1, color: "CCCCCC")
  - Cell margins: { top: 80, bottom: 80, left: 120, right: 120 }
- If Key Terms table is empty or absent, omit this section

**Section 6: Assembly and Output**
Define the full assembly steps:
1. Build all sections array in order (5a through 5i)
2. Create `new Document({ features: { updateFields: true }, styles: bookStyles, sections: allSections })`
3. Pack: `Packer.toBuffer(doc).then(buffer => fs.writeFileSync(outputPath, buffer))`
4. Output path: `[project_directory]/output/[Book Title].docx` (title from book-dna metadata, spaces preserved)
5. Create the `output/` directory if it does not exist

**Section 7: Generation Script Template**
Provide the complete Node.js script template that the formatter writes to a temp file and executes. The script:
- Requires docx module with all needed imports: Document, Packer, Paragraph, TextRun, Table, TableRow, TableCell, Header, Footer, AlignmentType, HeadingLevel, NumberFormat, PageNumber, TableOfContents, BorderStyle, WidthType, ShadingType, LevelFormat, TabStopType, PositionalTab, PositionalTabAlignment, PositionalTabRelativeTo, PositionalTabLeader
- Requires fs and path modules
- Reads project files (book-dna.md, all edited chapters)
- Builds all sections
- Generates the .docx
- Logs completion message with file path and size

**Section 8: Post-Generation**
After the script runs:
1. Verify the .docx file exists and is > 0 bytes
